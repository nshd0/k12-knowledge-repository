# telemetry/

**Purpose:** Usage metrics, workflow run logs, and observability data for monitoring system health and content coverage.

This folder stores telemetry outputs that help maintainers track how the system is performing, which content is most accessed, and where gaps exist.

## Rules

- Telemetry data must not contain personal data or user-identifying information
- Do not store raw logs here — only summarised, anonymised metrics
- Telemetry reports are auto-generated and should not be edited manually
