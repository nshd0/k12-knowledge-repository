# docs/

**Purpose:** Operational documentation, automated audit reports, deployment logs, and governance records.

This folder is written to by automated workflows. Do not manually edit audit reports — they are regenerated each cycle. Human-authored operational documents (runbooks, ADRs) are welcome here.

## Key files

| File | Contents |
|---|---|
| `monthly_audit_report.md` | Latest monthly content audit output (auto-generated) |

## Rules

- Audit reports are auto-generated by `monthly-review.yml` — do not edit them manually
- Runbooks and architecture decision records can be added here by contributors
- Do not store knowledge summaries here — those belong in `knowledge/`
