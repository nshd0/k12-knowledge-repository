# exports/

**Purpose:** Generated bundles for NotebookLM and other downstream consumers, produced from the cleaned `knowledge/` content.

All files here are auto-generated by `weekly-update.yml`. Do not edit them directly. If content is wrong, fix the source file in `knowledge/` or `data/`, then let the workflow regenerate exports on the next run.

## Rules

- Generated by workflow — do not edit manually
- Each export bundle corresponds to a stakeholder role or domain
- Outdated bundles are replaced automatically on each weekly refresh
- To trigger a manual regeneration, run `weekly-update.yml` via workflow dispatch
